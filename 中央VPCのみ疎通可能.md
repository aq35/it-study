## TGWルートテーブル設計（ポケモン版）

---

### 要件

> - 本番VPC ↔ 非本番VPC：**通信禁止**
> - 本番VPC・非本番VPC ↔ 共有サービスVPC：**通信OK**

---

### 登場人物

| 役割 | ポケモンで |
|------|-----------|
| TGWルートテーブル | **駅の案内板** |
| 本番VPC | **チャンピオンロード** |
| 非本番VPC | **サファリゾーン** |
| 共有サービスVPC | **ポケモンセンター（共用）** |

---

### ✅ 正解（選択肢2）の構造

**案内板A**（チャンピオンロード・サファリゾーンが登録）
```
掲示内容：「ポケモンセンターへの行き方のみ」
→ チャンピオンロードからはポケモンセンターにだけ行ける
→ サファリゾーンからもポケモンセンターにだけ行ける
→ お互いの行き方は載っていない＝直接行けない ✅
```

**案内板B**（ポケモンセンターが登録）
```
掲示内容：「チャンピオンロードとサファリゾーン両方への行き方」
→ ポケモンセンターから両方に行ける ✅
```

---

### ❌ 選択肢1「案内板Aに全ルート伝播」

```
案内板A掲示内容：
「ポケモンセンターへの行き方」
「サファリゾーンへの行き方」← これがあるとチャンピオンロードがサファリに行けてしまう
「チャンピオンロードへの行き方」← これがあるとサファリがチャンピオンロードに行けてしまう
```

> **禁止されている相互通信が可能になってしまう** ❌

---

### ❌ 選択肢3「共有サービスVPCを2つの案内板に登録」

```
案内板A：ポケモンセンターも登録
案内板B：ポケモンセンターも登録
```

> **「1つのアタッチメントは1つの案内板にしか登録できない」ルール違反**  
> ポケモンセンターは案内板Aか案内板Bのどちらか一方にしか登録できない ❌

---

### ❌ 選択肢4「案内板Aのルート伝播を無効」

```
案内板A掲示内容：（空白）
→ チャンピオンロードもサファリゾーンも
  どこにも行き方がわからない
→ ポケモンセンターにすら行けない ❌
```

> **案内板に何も書かれていない状態**  
> 通信禁止どころか**全部禁止**になってしまう

---

### 正解の設計を一言で

> **「チャンピオンロードとサファリゾーンの案内板には『ポケモンセンターへの道』だけ載せる。ポケモンセンターの案内板には『両方への道』を載せる。これで相互通信せず共有施設だけ使える。」**
